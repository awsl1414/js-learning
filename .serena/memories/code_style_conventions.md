# 代码风格和约定

## TypeScript 配置规范
- **目标标准**: ESNext (ES2025)
- **模块系统**: ES Modules (type: "module")
- **严格模式**: 启用 (`"strict": true`)
- **JSX 支持**: 启用 (`"jsx": "react-jsx"`)
- **JavaScript 兼容**: 允许 (`"allowJs": true`)

## 代码约定

### 变量和函数命名
- 使用 camelCase 命名变量和函数
- 使用 PascalCase 命名类和接口
- 使用 UPPER_CASE 命名常量
- 优先使用有意义的变量名

### 类型注解
- 所有函数参数应该有类型注解
- 函数返回值应该有类型注解（除非可以推断）
- 使用 interface 定义对象结构
- 使用 type 定义联合类型或复杂类型

### 模块和导入
- 使用 ES6 import/export 语法
- 优先使用具名导入和导出
- 避免使用 any 类型
- 合理使用泛型

### 代码结构
- 每个函数应该只做一件事
- 保持函数简洁（通常不超过 20 行）
- 使用描述性的函数名
- 添加必要的注释

### ES2025 特性使用规范
- 可以使用 ES2025 新特性
- 禁止使用已弃用的语法和特性
- 优先使用现代 JavaScript 语法
- 保持代码与最新标准兼容

### 错误处理
- 使用 try/catch 处理可能抛出的错误
- 提供有意义的错误信息
- 优雅地处理异步操作错误

### 性能考虑
- 避免不必要的全局变量
- 合理使用缓存
- 优化循环和递归
- 使用适当的异步模式

## 文件组织
- TypeScript 文件使用 .ts 扩展名
- JSX 文件使用 .tsx 扩展名
- 测试文件使用 .test.ts 或 .spec.ts 扩展名
- 保持文件职责单一

## 注释规范
- 使用 JSDoc 格式的注释
- 为复杂逻辑添加解释性注释
- 保持注释简洁明了
- 注释应该解释"为什么"而不是"是什么"
